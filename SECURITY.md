# Security Policy

## Supported versions

Security fixes are made on the current `main` branch. Release support windows
will be documented here when awesoMux publishes versioned releases.

## Reporting a vulnerability

Please do not report security vulnerabilities in public GitHub issues.
Instead, email [contact@interactivebuffoonery.com](mailto:contact@interactivebuffoonery.com).
GitHub's private vulnerability-reporting flow may also be available for this
repository; use it if it is enabled. Include the affected version or commit,
macOS version, a minimal reproduction, impact assessment, and any proposed
mitigation.

We will acknowledge a report, investigate it, and coordinate a fix before
public disclosure. Please give maintainers a reasonable opportunity to address
the issue before publishing details.
